# リリースノート v2.4.0

DESIGN_rev19に対応し、設定ファイルの管理性向上と機能別AIプロバイダの指定が可能になりました。

## 主な変更点

### 1. 設定管理の改善
- **リアクションルールの分離**: リアクションルールの定義を `config/reaction_rules.yaml` に分離しました。
- **.example運用への移行**: プリセット設定ファイルを `*.yaml.example` としてGit管理し、ユーザー環境での意図しない上書きを防止します。
- **セットアップスクリプト**: 雛形ファイルを一括コピーする `scripts/setup.sh` を追加しました。

### 2. 生成 AI 機能の強化
- **機能別プロバイダ指定**: リプライ、占い、タイムライン投稿、アンケートごとに個別のAIプロバイダ（Gemini / Ollama / OpenRouter）を指定できるようになりました。これにより、無料枠の使い分けなどが可能です。
- **投稿情報へのプロバイダ記録**: どのAIプロバイダを使用して投稿したかがデータベースに記録されるようになりました。

### 3. 管理機能の刷新
- **管理コマンドステータスの刷新**: `/admin status` の表示を絵文字ベースの簡潔なものに変更し、一目で状況が把握できるようになりました。
- **起動時チェック**: 必須設定ファイルが不足している場合にエラーを出力して起動を中断する機能を追加しました。

## アップデート時の注意
本バージョンから `config/serif/` 以下のファイル構成が変更されています。アップデート後は以下のコマンドを実行して設定ファイルを展開してください。
```bash
bash scripts/setup.sh
```
